Welcome to Scrappy!

Welcome to Scrappy, the client-sided only web scraper!
If you have any issues, please read the FAQ section in the about tab!
Here is a quick rundown what each tab does!

Search:

This is to scrape HTML objects and text based on your search criteria!
The HTML Parser is very similar to JQuery Syntax. Use the Cheerio API to reference the selectors.
Use the textboxs to fill out what you want to find.
Here are given examples (without the quotation):

Finding a tag:
Ex.) "p", "br", "h1", "h2", "h3", etc...

Finding a certain class:
Ex.) ".my-class-name", ".findMeThisClass", etc...

Finding attributes in a tag:
This is abit tricky. If you get undefined, this means that no such attribute does not exist or such a attribute has not been set within the tag. Refer to JQuery use of attr().

Ex.) Suppose we have the following html tag:

[class="hello" tabid="goodbye" style="..." ...]

First inputbox --> .hello
Second inputbox (the attribute text box) --> tabid

This will return "goodbye"

Common Elements:

***Scrapping May take time due to multiple HTML Elements for longer webpages! ***

This is to scrape HTML objects that are commonly found amongst webpages. Here is a rundown list of all the tags and their attributes that we can examine:

Links:
1.) [a][href="..."]
2.) [links][href="..."]

Text:
1.) [span]
2.) [p]
3.) [h1, h2, h3, h4, h5, h6]
4.) [b]
5.) [i]

Table:
1.) [tr]
2.) [th]
3.) [td]

List:
1.) [ul]
2.) [ol]
3.) [li]

Misc:
1.) [div]
2.) [img][src="..."]
3.) [img][alt="..."]

Navigate to the Common Elements and hit "Scrap!" to scrap the current URL that your DOM is focusing on (i.e the tab you are on)!

FAQ.

How come some of my elements are not showing up?

If you checked that your search parameters are correct in terms of JQuery Syntax, then the issue lies in CORS policy. The Cross-Origin Resource Sharing (CORS) it a HTTP-header mechanism that allows the server to identify the origins. Only selective html content is loaded due to CORS policy. Sadly, we cannot get around this (like disabeling CORS) The main issue is that the Chrome has CORS's policy enabled on the client, and disabeling will requires to disable web security which violates Chrome's Terms of Service. Sorry :(

Why are some list elements blank?

This is due to either the parser unable to find an innerHTML or your attributes/tags do not exist and JQuery cannot find it. Try to re-adjust or use inpect element to analyze the proper JQuery. Refer to

How is this useful? This seems very simple to me.

This is a client-sided webscraper to do quick look-ups and package results into Google Products or .json. Traditionally, you would need a server to do a heavy webscraper like Puppeteer or Phantom for node.js which uses a headless browser. However, this is designed to be lightweight for quick use. If you would like to contribute to add more features, please let me know!

Why am I recieveing the errors for unable to access: "chrome://..."

Due to Chrome's policy in accessing internal addresses, we cannot access such addresses that have "chrome://..." within it.
